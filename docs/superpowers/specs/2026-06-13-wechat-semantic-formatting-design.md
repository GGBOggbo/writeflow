# 公众号语义排版设计

## 目标

在正文生成完成后增加独立的“公众号排版”能力。用户可以选择原始版或去 AI 版正文，生成可预览、可复制到公众号后台的富文本 HTML。排版不得修改正文内容。

## 用户流程

1. 用户在正文阶段选择“原始版”或“去 AI 版”。
2. 点击“公众号排版，消耗 1 积分”。
3. 系统调用 AI，只把正文划分为语义块，不让 AI 生成 CSS 或改写正文。
4. 系统使用固定组件渲染 HTML，并默认展示“专业蓝”主题。
5. 用户可以免费切换“专业蓝”“温暖橙”“清爽青绿”主题。
6. 用户确认预览后，一键复制富文本，再粘贴到公众号后台。

一次正文版本的语义分析只扣 1 积分。切换主题、重复预览和重复复制不再次扣分。用户换了另一个正文版本后重新排版，视为新的付费操作。

## 架构

采用“AI 识别语义 + 程序稳定渲染”的混合方案。

AI 只返回结构化语义块：

- paragraph：普通段落
- heading：章节标题
- quote：核心金句
- pain：痛点或警示
- transition：转折或停顿
- list：短列表
- comparison：对比信息
- cta：结尾号召

每个块保留原文文本和原始顺序。服务端必须校验所有输出文本能按顺序还原为输入正文，防止模型增删或改写正文。校验失败时不渲染 AI 结果，退回保守段落解析。

程序根据语义块和主题 token 生成全部内联样式 HTML。AI 不接触主题色、CSS、HTML 标签或公众号兼容性逻辑。

## 主题

首发提供三套主题：

- professional-blue：专业蓝，默认主题，适合科技、商业、AI 和方法论。
- warm-orange：温暖橙，适合个人 IP、经历和成长复盘。
- fresh-teal：清爽青绿，适合知识型和个人品牌内容。

主题只改变颜色、圆角、边框和标题表现，不改变语义结构和正文。主题切换完全在前端或本地渲染完成，不调用 AI。

## 兼容性边界

生成结果只使用公众号编辑器更容易保留的基础内联样式：

- 字号、字重、行高、文字颜色
- 背景色、边框、圆角
- 内外边距、文本对齐
- 单列块级布局

首发不使用外部 CSS、style 标签、JavaScript、复杂定位、动画、CSS Grid、复杂 Flex 布局、外部字体和未经验证的渐变效果。

正文容器以手机阅读为主，最大宽度 640px，正文 15px，行高约 1.8。复制时只复制文章内容区域，不复制页面预览外壳。

## 数据与状态

排版结果与正文版本绑定，至少保存：

- draftVersionId
- semanticBlocks
- selectedTheme
- generatedAt

工作流状态保存语义块，不保存三份重复 HTML。HTML 根据语义块和主题即时渲染，避免主题切换产生重复数据。

## 积分与接口

新增积分阶段 format，沿用现有 operation ID、预占、提交和失败退款机制。

新增独立流式接口 /api/ai/format/stream：

- 输入：正文版本 ID、正文内容
- 输出：结构化语义块
- 成功后扣 1 积分
- AI 调用或校验失败时不消耗积分

主题切换和 HTML 渲染不经过该接口，因此不扣积分。

## 前端交互

正文阶段增加“公众号排版”入口。生成后打开排版面板：

- 左侧或顶部选择主题
- 中间展示手机宽度预览
- 提供“一键复制富文本”
- 显示当前排版对应的正文版本
- 切换正文版本时提示需要重新生成排版

首发不提供自由调色、字号滑杆或任意 CSS 编辑，避免功能和兼容性失控。

## 错误处理

- AI 语义识别失败：退还积分，正文保持不变，可重试。
- AI 改写正文：校验失败并退还积分，不使用该结果。
- 浏览器不支持富文本复制：降级复制 HTML 或提示用户打开预览页面全选复制。
- 主题渲染异常：退回专业蓝的保守样式，不重新调用 AI。
- 本地恢复时语义块缺失：显示“重新生成排版”，不影响正文工作流。

## 测试

- Prompt 测试：明确禁止改写正文和输出 HTML/CSS。
- Schema 测试：只接受允许的语义块类型。
- 完整性测试：语义块按顺序还原后与输入正文一致。
- 渲染测试：三套主题输出均为内联样式且不含 style/script。
- 积分测试：首次生成扣 1 分，切换主题不扣分，失败退款。
- 状态测试：排版结果与正确正文版本绑定并可恢复。
- UI 测试：主题切换、预览和复制入口可用。
- 浏览器验证：实际复制到可编辑富文本区域，确认标题、金句、痛点框和段落间距保留。

## 非目标

首发不做 AI 自由生成整篇 HTML，不做图片上传或自动配图，不做公众号后台自动发布，不做模板市场，不在排版步骤再次润色正文。
