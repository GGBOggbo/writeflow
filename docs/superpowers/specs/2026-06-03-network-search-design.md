# AI Writing MVP 联网搜索设计说明

日期：2026-06-03

## 目标

在不破坏现有主流程和 IP 真实感的前提下，为 AI 写作 MVP 增加“联网增强”能力。

核心原则：

1. 找痛点、找网感、找最新表达时联网。
2. 写正文时断网，保护人设、经验和真实感。
3. 搜索失败不能阻塞主流程，必须能自动降级。

## 设计结论

采用“首尾默认联网，中段按需联网，正文永久断网”的策略。

阶段规则如下：

| 阶段 | 默认搜索策略 | 是否允许手动开关 | 说明 |
|------|--------------|------------------|------|
| Idea | 关闭 | 否 | 用户原始想法，不需要外部信息污染 |
| Topics | 开启 | 是 | 依赖最新热点、痛点词、情绪词、平台语境 |
| Brief | 关闭 | 是 | 默认保护 IP 方向；卡住时可联网找灵感 |
| Outline | 关闭 | 是 | 默认保护骨架独特性；可按需参考对标结构 |
| Draft | 强制关闭 | 否 | 正文必须基于大纲与真实素材闭门组装 |
| Meta | 开启 | 是 | 依赖最新标题网感、社交货币和包装表达 |

## 产品规则

### 1. 联网增强不是全局无脑搜索

联网搜索只作为“参考语料输入”，不直接成为正文内容。

允许联网的阶段，搜索结果只可用于：

- 选题切口参考
- 热点和情绪词参考
- 标题句式参考
- 平台网感参考
- 对标结构灵感参考

严禁将联网结果直接伪装成：

- 用户亲身经历
- 客户案例
- 第一手业务数据
- 正文中的“我做过”“我踩过坑”类内容

### 2. Draft 阶段物理隔离

正文阶段不提供联网开关，不读取搜索结果，不消费搜索上下文。

原因：

- 避免洗稿式拼接
- 避免稀释 IP 人设
- 避免正文失去真实感
- 避免把“搜索最新信息”误用成“正文自动引用外部内容”

### 3. 手动开关的产品含义

手动开关代表“是否允许当前阶段参考最新网页信息”，不是“让 AI 自动联网写作”。

用户拥有最终判断权：

- 开：参考最新语料
- 关：仅基于当前稿件上下文生成

## UI 设计

### 1. 全局展示方式

在工作区顶部或阶段操作区加入轻量开关：

- 文案：`联网增强`
- 辅助说明：`参考最新网页信息，速度会变慢`

### 2. 默认行为

进入不同阶段时，开关默认值按阶段自动设定：

- Topics：开
- Brief：关
- Outline：关
- Draft：强制关且不可操作
- Meta：开

### 3. Draft 阶段的交互

正文阶段显示断网说明，而不是显示可点击开关：

- 文案：`正文阶段默认不联网，以保留真实经验和人设表达`

### 4. 搜索失败的降级提示

如果联网搜索失败，不中断当前生成，只提示：

- `联网搜索暂时不可用，已切回基础生成`

## 工程设计

### 1. 新增独立搜索层

不要把搜索能力直接混进 provider。

分层建议：

- `AI provider`：负责生成
- `search service`：负责检索最新网页信息
- `prompt layer`：决定是否把搜索结果拼入当前阶段 prompt

推荐新增结构：

- `lib/search/types.ts`
- `lib/search/service.ts`
- `lib/search/provider.ts`
- `lib/search/mock-provider.ts` 或 fallback 逻辑
- `app/api/search/...`（如果搜索能力单独提供接口）

### 1.1 SearchResult 不能只做通用搜索结构

第一版搜索结果结构不能停留在通用搜索引擎的：

- `title`
- `snippet`
- `url`
- `publishedAt`
- `source`

还必须补上“爆款验证”和“搜索运营”维度。

推荐结构：

```ts
type SearchResult = {
  title: string;
  snippet: string;
  url: string;
  publishedAt?: string;
  source: "wechat" | "xiaohongshu" | "hotlist" | "generic";
  engagementMetrics?: {
    readCount?: number;
    likeCount?: number;
    collectCount?: number;
    commentCount?: number;
  };
  seoKeywords?: string[];
  crowdedness?: "low" | "medium" | "high";
  staleBuzzwords?: string[];
  notes?: string[];
};
```

其中关键新增字段含义：

- `engagementMetrics`：判断内容是否是“被验证过的爆款”
- `seoKeywords`：供标题/小标题/正文结构自然埋词
- `crowdedness`：判断话题是否过挤
- `staleBuzzwords`：识别已烂掉的行业黑话
- `notes`：向 prompt 传递“避免跟风”“换新说法”等策略提示

### 2. 阶段生成接口增加搜索配置

对允许联网的阶段，在请求中增加显式参数：

- `searchEnabled: boolean`
- `searchMode: "default" | "manual"`

如后续需要更细控制，可扩展：

- `searchIntent: "topics" | "brief" | "outline" | "meta"`

### 3. Prompt 接入策略

只在以下阶段消费搜索结果：

- `topics`
- `brief`（可选，手动开启时）
- `outline`（可选，手动开启时）
- `meta`

Prompt 中新增一段“最新参考信息”，但必须明确说明：

- 这些信息仅供参考
- 不可编造事实
- 不可把搜索结果伪装为第一手经历

并补上 3 条搜索后的硬规则：

1. 优先参考高互动结果，低互动或无验证结果只做弱参考。
2. 如果话题过挤或关键词已烂，要明确提示模型“意思不变，换一种用户没见过的说法”。
3. 搜索阶段产出的 `seoKeywords` 需要单独回填给 `topics` / `outline` / `meta`，用于标题与结构的自然埋词，不直接灌进正文内容。

### 4. 状态管理

搜索开关应进入前端 workflow 状态或独立 UI state，至少支持：

- 当前阶段开关值
- 默认阶段映射
- Draft 阶段锁定不可开启

推荐新增一个轻量设置结构：

```ts
type SearchSettings = {
  topics: boolean;
  brief: boolean;
  outline: boolean;
  meta: boolean;
};
```

Draft 不进入该结构，直接产品层面禁用。

## API 与数据流

### Topics

数据流：

`idea -> optional search -> topics prompt -> AI provider`

搜索目标：

- 痛点词
- 热门表达
- 情绪词
- 平台长尾语境
- 已验证的高互动爆款切口
- 可自然埋入标题/小标题的 SEO 长尾词

搜索源优先级：

1. `wechat`：同赛道公众号文章、近半年、热度优先
2. `xiaohongshu`：下拉词、评论区、痛点表达
3. `hotlist`：热榜/热搜，用于“热点 + 观点”拼接

输出给 prompt 的不只是“灵感语料”，还包括：

- 高互动参考结果
- 推荐的 SEO 长尾关键词
- 过挤话题预警
- 烂词替换提醒

### Brief / Outline

数据流：

`current stage input -> optional manual search -> prompt -> AI provider`

搜索目标：

- 用户吐槽点
- 对标结构
- 场景表达
- 小标题可埋的搜索关键词
- 可借鉴但不能照抄的爆款结构

默认关闭，用户手动开启后才参与。

如果开启搜索，返回内容除了普通结果，还要包含：

- `seoKeywords`
- `crowdedness`
- `staleBuzzwords`

### Draft

数据流：

`brief + outline + material slots -> draft prompt -> AI provider`

无搜索参与。

### Meta

数据流：

`draft content + persona + dropOffPoint + optional search -> meta prompt -> AI provider`

搜索目标：

- 最新标题风向
- 热门句式
- 平台语境
- 社交货币表达
- 高互动标题模型
- 过期黑话过滤
- 搜索流量友好的长尾关键词

搜索源优先级：

1. `wechat`：同类高阅读标题
2. `xiaohongshu`：用户真实情绪表达与点击语言
3. `hotlist`：最新趋势词与新鲜表达

meta prompt 必须显式消费：

- 高互动标题样本
- SEO 长尾关键词
- 黑话/烂词警告
- 过挤话题警告

并要求模型：

- 优先参考高互动标题
- 避免使用过度泛滥词汇
- 若发现表达已烂，执行“同义换新说法”

## 搜索源策略

### 不推荐只用通用搜索引擎

单纯接 Bing / Google / 百度式 provider，会过度返回：

- 官网
- 百科
- 新闻资讯
- 低情绪密度内容

这不适合爆款选题和标题包装。

### 推荐的第一版渠道优先级

1. `wechat`
   - 用于找同赛道公众号爆款
   - 重点看文章热度、发布时间、标题风格
2. `xiaohongshu`
   - 用于抓长尾需求词、评论区痛点、下拉词
   - 重点看情绪表达和真实需求
3. `hotlist`
   - 用于热点 + 垂直切口拼接
   - 重点看趋势变化和新鲜语境

如果第一版因数据源限制必须先接通用搜索，也应在 provider 层保留 source abstraction，避免未来难以切换。

## SEO 关键词回填

Search service 不能只返回“参考文章”。

它还必须额外提炼：

- 高搜索量、低竞争的长尾关键词
- 适合标题使用的关键词
- 适合小标题使用的关键词

这些关键词只用于：

- `topics`：帮助选题更接近用户真实搜索语言
- `outline`：自然进入大纲标题和节点命名
- `meta`：帮助标题摘要兼顾点击率和搜索权重

禁止要求 `draft` 强行堆砌 SEO 词频，但允许在结构与小标题层提前种下关键词。

## 过挤话题与烂词过滤

搜索层必须在清洗阶段做两种过滤：

### 1. 过挤话题检测

如果某个话题近期高频出现、表达高度同质、互动已下滑，应标记：

- `crowdedness: high`

并向 prompt 注入提醒：

- `此话题已被过度讨论，请避免跟风复述，改用新角度或新表述。`

### 2. 黑话/烂词过滤

若搜索结果中频繁出现这类高麻痹词：

- 杠杆
- 底层逻辑
- 护城河
- 轻创业
- 一人公司

则标记进：

- `staleBuzzwords`

并要求 `topics` / `meta` prompt 执行：

- `意思不变，换一种更具体、更新鲜、更接地气的大白话表达。`

## 错误处理与降级

### 1. 搜索失败

搜索失败时：

- 不报整页错误
- 不中断 AI 生成
- 自动退回无搜索生成

### 2. 搜索超时

搜索请求应设置独立超时，避免拖死生成链路。

建议：

- 搜索超时短于主生成超时
- 超时后直接降级

### 3. 搜索结果为空

若未取到有效结果：

- 继续基础生成
- 不显示硬错误

## 测试策略

### 单元测试

覆盖：

- 阶段默认搜索开关映射
- Draft 阶段禁止联网
- 搜索失败自动降级
- 搜索参数正确传给 topics / brief / outline / meta
- Draft 请求中没有搜索参数或搜索上下文
- `SearchResult` 结构包含 engagement / seo / crowdedness / staleBuzzwords
- 高互动结果排序优先
- 黑话过滤提醒是否正确生成
- 过挤话题是否注入告警
- SEO 关键词是否正确回填到 topics / outline / meta 的 prompt builder

### 集成测试

覆盖：

- Topics 默认联网
- Meta 默认联网
- Brief / Outline 手动开启后参与搜索
- 搜索失败时仍能完成生成

### 浏览器回归

重点验证：

- 不同阶段开关状态是否正确
- Draft 是否真的不可开启联网
- 切换开关后生成行为是否符合预期

## 非目标

这一版不做：

- 正文联网写作
- 搜索结果来源展示 UI
- 用户手动编辑搜索 query
- 搜索证据引用卡片
- 多搜索引擎聚合排序

## 已确认但暂缓的增强项

以下能力已确认有价值，但不进入当前“联网搜索”实现范围，只记录为后续产品增强：

### 1. IP Core Belief

在 `Persona` 之外补一个更底层的字段：

- `Core Belief / 核心信仰`

作用：

- 让账号长期输出拥有稳定价值主线
- 让正文、标题、结尾都最终回归同一根“信念针”

### 2. 标题黑话过滤器

在 `meta` 包装阶段单独增加一层规则：

- 过滤已烂掉的行业词
- 避免“看起来专业，实际上用户已经麻痹”的表达

这项能力未来应与搜索层的：

- `staleBuzzwords`

联动，但本轮先不独立实现产品化开关。

### 3. 封面图建议

在标题摘要之外，补一条：

- `封面图配图建议`

输出维度可包括：

- 推荐封面类型
- 图上大字建议
- 是否建议复用真人视觉资产
- 是否应突出工牌 / 场景 / 人脸 / 工作照

## 推荐实施顺序

1. 先做 `topics` 搜索增强
2. 再做 `meta` 搜索增强
3. 然后补 `brief` / `outline` 的手动联网增强
4. 最后再考虑更复杂的搜索来源展示和引用策略

## 验收标准

1. Topics 默认参考最新搜索结果。
2. Meta 默认参考最新搜索结果。
3. Brief / Outline 默认不搜索，但可手动开启。
4. Draft 完全不联网，且无手动入口。
5. 搜索失败不阻塞主流程，能自动降级。
6. 前端能清楚表达“联网增强”的作用与限制。
