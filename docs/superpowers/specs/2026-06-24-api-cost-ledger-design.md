# API 成本账本设计

日期：2026-06-24

## 目标

在项目内建立一份长期可查、可维护的 API 成本账本，避免每次重新检查代码和手工计算。

账本需要同时回答：

- 用户在产品内会被扣多少积分。
- 后端实际调用了哪些外部接口。
- 每条调用路径最多产生多少 wxrank 成本。
- 哪些成本是固定值，哪些会随实际请求变化。
- AI 模型成本应如何记录和计算。

## 交付物

新增 `docs/api-cost-ledger.md`，并在 `README.md` 的联网搜索说明附近添加入口。

本次只建立项目内文档，不开发管理后台、数据库统计或自动计费功能。

## 成本口径

### 产品积分

产品当前对 `topics`、`brief`、`outline`、`draft`、`meta` 五个生成阶段分别收取 1 项目积分。生成失败时退款。

项目积分是产品额度，不直接等于人民币，也不等于 wxrank 积分。

### wxrank 单价

以 2026-06-24 用户提供的 wxrank 价格说明为准：

| 接口 | 单价 |
|---|---:|
| `artlist` | ¥0.01/次 |
| `getso` | ¥0.10/次 |
| `artinfo` | ¥0.01/次 |
| `getcm` | ¥0.05/次 |
| `score` | 免费 |

账本也应列出项目暂未使用的 wxrank 接口单价，方便未来接入时查阅，但必须标记为“当前流程未调用”。

### 当前公众号搜索路径

当前默认配置为：

- 合格历史结果阈值：5 篇。
- 最终结果上限：8 篇。
- 深拆文章上限：4 篇。
- 留言文章上限：2 篇。
- 每篇保留留言上限：10 条。

公众号搜索成本表：

| 路径 | 调用组成 | 项目扣分 | wxrank 成本上限 |
|---|---|---:|---:|
| 当月历史命中 | `artlist × 1 + artinfo × 4 + getcm × 2` | 1 | ¥0.15 |
| 补查上月命中 | `artlist × 2 + artinfo × 4 + getcm × 2` | 1 | ¥0.16 |
| 实时搜索兜底 | `artlist × 2 + getso × 1 + artinfo × 4 + getcm × 2` | 1 | ¥0.26 |
| 未开启联网搜索 | 无 wxrank 请求 | 1 | ¥0.00 |

`getcm` 仅在文章存在 `comment_id` 时调用，因此表内金额是调用上限：

- 当月历史命中的实际区间为 ¥0.05～¥0.15。
- 补查上月命中的实际区间为 ¥0.06～¥0.16。
- 实时搜索兜底的实际区间为 ¥0.16～¥0.26。

后续 Brief、大纲、正文和标题摘要复用选题阶段的搜索上下文，不重复调用 wxrank。

## AI 成本

AI 成本与 wxrank 成本分开记录。

账本应列出每个阶段可能发生的 AI 请求，例如搜索规划、标杆总结、选题、Brief、大纲、正文和标题摘要，并说明：

- 产品侧每次成功生成扣 1 项目积分。
- 一个产品操作内部可能包含多次 AI 请求。
- 人民币成本取决于模型输入 Token、输出 Token 和供应商实时单价。
- 在没有稳定 Token 用量统计与供应商价格配置前，不填写虚构的固定金额。

账本提供以下计算公式：

`AI 成本 = 输入 Token ÷ 1,000,000 × 输入单价 + 输出 Token ÷ 1,000,000 × 输出单价`

当前 `.env.local` 的模型选择只作为本地环境备注，不写入公开账本，以免将个人环境配置误认为项目默认值。

## 维护规则

为了降低文档过期风险：

1. 成本表必须注明核对日期。
2. 调用次数以当前代码和默认环境变量为准。
3. 单价以供应商价格页或用户提供的正式价格说明为准。
4. 修改以下配置或调用链时，应同步更新账本：
   - `WXRANK_HISTORY_MIN_RESULTS`
   - `WXRANK_RESULT_LIMIT`
   - `WXRANK_DEEP_DIVE_LIMIT`
   - `WXRANK_COMMENT_ARTICLE_LIMIT`
   - wxrank 搜索路由
   - AI 阶段或模型调用次数
5. README 只保留摘要和账本链接，完整成本只维护一份。
6. 旧设计文档中的历史估算不修改，但新账本需要注明旧数字已失效，避免误用。

## 验收标准

- 能在一分钟内查到一次公众号选题最多产生多少 wxrank 成本。
- 清楚区分项目积分、wxrank 人民币成本和 AI Token 成本。
- 表格与当前 `artinfo × 4`、`getcm × 2` 的代码默认值一致。
- README 能直接进入成本账本。
- 不把项目积分写成固定人民币价值。
- 不虚构尚未统计的 AI 实际成本。
