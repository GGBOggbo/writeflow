# 草稿阶段预览布局重组

## 问题证据

`DraftStage`（`components/stages/draft-stage.tsx`）当前把排版面板当作一个独立整块，排在「版本管理 / 正文内容」并排区下方（`<section className="space-y-6">` 垂直堆叠）。结果：版本与正文在上层左右并排，而公众号排版预览被压到最下方，与上方并排内容在视觉上割裂，用户必须大幅下滚才能看到排版效果。

## 目标

- 排版预览与正文同层并排（对读），不再独占下方整块。
- 版本管理从左侧 260px 列表上移为正文上方的横向 tab，释放横向空间给「正文 / 预览」并排。
- 保留生成前 / 生成后两种状态、响应式降级，以及现有交互（切换版本、切换主题、复制、积分扣费）。

## 目标布局

### 生成后（xl ≥ 1280px）

```
┌───────────────────────────────────────────┐
│ 第五步  版本对读                              │
├───────────────────────────────────────────┤
│ [ 原稿 ] [ 润色版 ] [ 去AI版 ]   ← 版本横向 tab │
├─────────────────────┬─────────────────────┤
│                     │   文章预览            │
│   正文内容           │   ┌───────────────┐  │
│   (ProseText)        │   │  公众号文章渲染  │  │
│                     │   └───────────────┘  │
│                     │  ●专业蓝 ●温暖橙 ●青绿 │
│                     │           [复制到公众号]│
└─────────────────────┴─────────────────────┘
[ 去 AI 味 ]   [ 生成标题和摘要 ]
```

### 生成前（右侧排版位显示入口，不空置）

```
┌─────────────────────┬─────────────────────┐
│                     │  公众号排版            │
│   正文内容           │  识别语义，套用稳定样式  │
│                     │ ┌─────────────────┐  │
│                     │ │ [公众号排版·1积分] │  │
│                     │ └─────────────────┘  │
└─────────────────────┴─────────────────────┘
```

### 响应式

- xl（≥1280px）：正文 | 排版预览 并排。
- <xl：堆叠降级（正文在上、排版在下）。
- 版本 tab 在所有断点都保持正文上方的横向形态。

## 具体改动

### components/stages/draft-stage.tsx

- 移除现有的 `xl:grid-cols-[260px_1fr]`（版本管理左栏 + 正文右栏）。
- 版本管理改为正文区上方的一行横向 tab 按钮（沿用 `handleSelectDraft` 与 `activeDraft` 高亮逻辑）。
- 新增主区并排容器：`xl:grid-cols-2`。左列 = 正文（`ProseText` 卡片），右列 = `WechatFormatPanel`。
- `WechatFormatPanel` 仍带 `key={activeDraft.id}`，排版结果随版本切换沿用 `state.draftFormattingByVersion`。
- 底部「去 AI 味」「生成标题和摘要」按钮保持不变。

### components/wechat-format-panel.tsx

**生成前**：

- 渲染入口卡片：简短标题 + 一行说明 + 「公众号排版，消耗 1 积分」按钮，沿用现有 `onGenerate` / `canGenerate` / `loading` / `disabled`。
- 不再占整行宽度；宽度跟随右侧位列宽。

**生成后**：

- 预览主体在上：公众号文章渲染（`dangerouslySetInnerHTML`），可滚动，保留 `max-h` 与卡片阴影。
- 底部控制条横向排布：主题色点选项（专业蓝 / 温暖橙 / 清爽青绿，沿用 `onThemeChange`）+ 复制按钮（沿用 `copyRichHtml`）+ 复制状态文案。
- 精简原头部的大段说明文字（并排后空间紧张）；「已完成语义识别」状态作为预览头部的小标签显示。

### components/wechat-format-panel.test.tsx

- 更新布局相关 DOM 断言以匹配新结构（生成前入口、生成后「预览 + 底部控制条」），保留必要的 `data-testid`。
- 保留行为断言：主题切换触发 `onThemeChange`、复制触发 `copyRichHtml`、积分不足按钮禁用。

## 默认决策（已确认）

1. 生成前右侧位 = 排版入口卡片（可直接点生成，不空置）。
2. 控制（主题 / 复制）= 预览下方横向条。
3. 正文 / 预览在 xl 各占 1fr；预览内部仍模拟手机宽度，贴近真实公众号观感。

## 不在本次范围

- `renderWechatHtml` / `themes.ts` / 微信兼容输出层（字体继承强制化已在另一轮处理）。
- 版本切换、积分扣费、复制的数据流与 handler。
- 底部「去 AI 味」「生成标题和摘要」按钮。

## 验收

- xl 屏：正文与排版预览左右并排，版本 tab 在正文上方横向；排版不再在最下方独占整块。
- 生成前：右侧位显示排版入口按钮。
- 生成后：右侧位为预览 + 底部控制条；切换主题、复制均正常，不重新调用 AI。
- <xl 屏：堆叠降级，版本 tab 仍横向顶部。
- `wechat-format-panel.test.tsx` 与相关组件测试全绿，无回归。
