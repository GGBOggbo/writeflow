# 公众号模块渲染器参考骨架对齐设计

## 背景

当前正文使用相同的 `:::` 模块语法，但存在两套不同的视觉契约：

- 项目本地 `advanced-module-render.ts` 使用绿色 `wechat-native` 简化骨架。
- 用户提供并已归档的标准 HTML 使用暖纸主题和更完整的模块骨架。

以 `cta` 为例，本地版本只渲染 `title + note`；标准 HTML 还包含短行动标题、三张固定操作卡和底部注释。当前 AI 又把一整段论述同时填入 `title` 与 `note`，导致浏览器中 CTA 高达 417px、正文重复、视觉割裂。

归档真源：

- `docs/audits/fixtures/md2wechat-wechat-native/full-module-source.md`
- `docs/audits/fixtures/md2wechat-wechat-native/full-module-rendered.html`

## 目标

1. 让本地 31 类高级模块的结构和暖纸视觉语言与归档标准 HTML 对齐。
2. 修复 AI 把普通论述或核心判断误判为 CTA 的问题。
3. 防止模块字段过长、字段重复和无效结构进入渲染阶段。
4. 让浏览器预览与复制到微信使用同一个渲染真源。
5. 保留微信复制阶段必要的确定性降级，不要求网页交互 CSS 原样进入微信。

## 非目标

- 不修改正文事实或自动重写用户文章。
- 不要求每篇文章使用全部 31 个模块。
- 不复制外部服务端代码；仅依据用户拥有并提供的 Markdown/HTML 输入输出样本实现确定性渲染。
- 不新增 CLI、远程 API或额外积分消耗。
- 不修改数据库、认证、搜索或积分业务。

## 方案选择

### 方案 A：只修当前 CTA

改动最小，但 `verdict` 和其他模块仍与参考 HTML 不一致，之后会继续逐个返工。

### 方案 B：整体对齐参考骨架（采用）

建立集中式暖纸 token 和模块视觉契约，先修 CTA、verdict 与语义边界，再以完整样稿覆盖全部模块。改动范围更大，但只有这一方案能消除两套渲染真源。

### 方案 C：运行时直接使用标准 HTML

无法处理用户动态正文，也无法保证安全清洗、字段校验和微信降级，因此不采用。

## 架构

### 1. 视觉真源

归档 HTML 是视觉回归样本，不作为运行时依赖。生产代码继续由 TypeScript 确定性生成 HTML。

新增或重构集中式主题 token，至少包含：

- 页面、正文、弱化文字和强调色
- 暖纸背景、边框、圆角和阴影
- 标题、正文、标签的字号与行高
- 模块外边距、内边距和子项间距
- CTA、verdict 等特殊模块参数

模块渲染器只能读取集中 token，避免 31 个函数各自散落颜色和尺寸。

### 2. 模块契约

`MODULE_DEFS` 继续作为字段真源，并补充内容约束：

- `cta.title`：短行动引导，不接收完整论述。
- `cta.note`：可选短注释，不能与标题重复。
- `verdict.title`：一句核心判断。
- `verdict.body`：对判断的简短解释。

校验层不截断用户文本。遇到不合格模块时，优先把内容安全降级为普通 Markdown，而不是生成畸形卡片。

### 3. AI 模块选择

排版 Prompt 明确区分：

- `cta`：只用于文末存在明确下一步行动的内容。
- `verdict`：用于核心结论或转折判断。
- 普通论述：保留为自然段，不为了视觉丰富强行模块化。

要求模型不得把同一段内容同时填入多个字段，不得把超过合理长度的论述塞入 CTA 标题。

本地后处理负责确定性兜底：

- CTA 不在文章末段或没有行动语义时，降级为普通正文或 verdict。
- `title` 与 `note` 高度重复时，删除重复 note。
- 字段明显超长时，不截字，整个模块降级以保留完整内容。

### 4. 渲染路径

统一数据流：

```text
Markdown + ::: 模块
  → parseAdvancedMarkdown
  → 模块契约校验/安全降级
  → renderExtendedMarkdown
  → 浏览器预览
  → normalizeWechatHtml
  → 微信复制
```

预览和复制不得使用两套不同的模块模板。`normalizeWechatHtml` 只负责微信不支持能力的降级，例如多列、横滑和长图滚动，不改变模块内容含义和主题视觉。

## CTA 目标骨架

合法 CTA 使用参考样本结构：

1. 顶部短行动标题。
2. 中部三张固定操作卡：保存灵感、直接套用、继续体验。
3. 底部可选短注释。
4. 暖纸渐变、低对比边框和橙棕强调色。

在手机预览宽度下保留参考样本的两列布局，第三张卡位于第二行。微信复制时若网格不稳定，兼容层将操作卡确定性拍平为纵向块，但保留内容、颜色和层级。

## Verdict 目标骨架

使用参考样本结构：

1. 小型强调标记和 eyebrow。
2. 一句高权重核心判断。
3. 一段较轻的解释。
4. 暖纸渐变背景，不使用大面积高饱和绿色。

## 错误处理

- 未知模块、字段缺失：保留为可编辑 Markdown，不静默输出空卡。
- 字段超长或重复：记录结构化诊断事件并降级，不丢正文。
- 非法 URL 或 HTML：沿用现有安全清洗。
- 微信不支持的布局：复制阶段拍平，浏览器预览仍可保留完整布局。

## 日志

排版完成日志增加：

- `moduleNames`
- `moduleCount`
- `degradedModules`
- `degradationReasons`
- `ctaCount`

不得记录完整正文或敏感素材。

## 测试

### 契约测试

- CTA 仅接受短行动内容。
- 重复 `title/note` 会被去重或降级。
- 普通论述不会被后处理强制转成 CTA。
- 核心判断优先使用 verdict。

### 渲染快照测试

- 31 种模块均存在独立结构断言。
- CTA 和 verdict 对照归档 HTML 的关键 DOM、颜色、间距和层级。
- 不对整个 156KB HTML 做脆弱的全文字符串快照。

### 集成测试

- Markdown 编辑后右侧即时使用新骨架。
- 复制输出和预览来自同一份渲染 HTML。
- 微信降级后没有 CSS grid/flex 依赖，也不丢模块正文。

### 浏览器回归

- 使用当前 Claude Code 长文验证误判 CTA 已消失。
- 使用完整 411 行样稿验证 31 类模块。
- 检查 411px 手机预览无横向溢出。
- 对比 CTA、verdict 和长文首屏的视觉密度。

## 实施顺序

1. 为 CTA 语义误判、字段重复和参考 DOM 写失败测试。
2. 修复 AI Prompt 与本地模块后处理。
3. 建立集中暖纸 token。
4. 对齐 CTA 与 verdict。
5. 分组对齐其余 29 个模块。
6. 统一预览/复制入口并验证微信降级。
7. 运行全量测试、lint、构建和浏览器视觉回归。

## 验收标准

- Claude Code 长文中的结构论述不再渲染成巨大 CTA。
- 合法 CTA 不包含重复正文，视觉与归档参考骨架一致。
- verdict 使用暖纸骨架且字段层次清晰。
- 31 类模块均通过结构测试。
- 浏览器预览与复制输出不存在两套主题模板。
- 全量测试、ESLint、Next.js 构建和 `git diff --check` 通过。
