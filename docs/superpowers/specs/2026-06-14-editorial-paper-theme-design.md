# 公众号“编辑纸感”双主题设计

## 目标

在现有“春日清新”排版之外新增“编辑纸感”，让用户对同一份语义排版结果免费切换视觉风格。新主题借鉴 Open Design 的 Magazine Article 与 Kami 排印逻辑，强调连续阅读、排印层级和克制装饰。

## 用户体验

- 公众号排版生成完成后，预览区提供“春日清新”和“编辑纸感”两个主题按钮。
- 主题切换只重新渲染本地 HTML，不调用 AI、不消耗积分。
- 当前选择保存到对应正文版本的 `DraftFormatting.selectedTheme`。
- 切换正文版本时恢复该版本上次选择的主题。
- 默认新生成的排版仍使用“春日清新”，避免改变现有用户习惯。

## 主题边界

### 春日清新

保留现有实现：浅灰绿画布、章节白卡、圆点纹理、花形标题、圆角阴影、彩色语义块和绿色渐变 CTA。

### 编辑纸感

- 画布使用暖纸色，正文使用暖深灰，唯一强调色为墨绿。
- 正文连续排列，不为每个章节套独立悬浮卡片。
- 章节通过字号、字重、留白和细分隔线建立层级。
- 不使用花形标题装饰、圆点卡片纹理、明显阴影或多色语义块。
- 金句使用墨绿左侧细线，不使用完整卡片阴影。
- 痛点提醒使用浅灰绿底和墨绿细线，不引入橙色。
- 列表与对比保持轻量边框或浅底，不形成层层嵌套卡片。
- 转折使用克制的中央 ornament 或细分隔线。
- CTA 只保留文末一个，以上下细线和墨绿文字强调，不使用渐变色块。
- 正文字号约 16px，行高约 1.85，段落间距约 20px；内容宽度继续受现有微信预览容器约束。

## 数据与兼容

- `WechatFormatTheme` 扩展为 `spring-fresh | editorial-paper`。
- `draftFormattingSchema` 接受两个正式主题，并继续把旧主题 `professional-blue`、`warm-orange`、`fresh-teal` 迁移为 `spring-fresh`。
- 主题切换恢复为纯状态事件，只更新 `selectedTheme`，不替换 `blocks`、`generatedAt` 或正文内容。
- 现有已保存的 `spring-fresh` 数据无需迁移。

## 渲染架构

- AI 语义分类结果 `FormattingBlock[]` 保持不变。
- `renderWechatHtml(blocks, theme)` 根据主题选择视觉渲染策略。
- 春日清新继续使用章节分组卡片。
- 编辑纸感使用连续文章容器：标题开启章节，但章节本身不增加卡片背景与阴影。
- 两个主题复用 Markdown 安全渲染、HTML 转义和微信兼容清理逻辑。
- 所有最终样式继续以内联 CSS 输出，确保复制到公众号编辑器后尽量保留。

## UI

- 在现有预览区底部恢复精简的主题切换控件，仅显示两个选项。
- 主题名为“春日清新”和“编辑纸感”。
- 当前主题有明确选中状态，但控件不喧宾夺主。
- 空状态文案说明首次生成消耗 1 积分，主题切换与复制免费。

## 测试与验收

- 渲染测试覆盖两个主题及全部语义块。
- 编辑纸感断言：无章节卡片纹理、无花标、无渐变 CTA，存在暖纸背景、墨绿分隔线和连续章节结构。
- 状态机测试覆盖切换主题不修改语义块。
- 存储测试覆盖两个正式主题恢复及三个旧主题迁移。
- 组件测试覆盖两个按钮、选中状态和切换回调。
- App 流程测试确认生成请求只发送一次，后续主题切换不增加网络请求。
- 运行全量测试、lint、生产构建，并在本地浏览器用真实长文目视验证两套效果。

## 非目标

- 不修改 AI 语义分类提示词。
- 不增加新的积分扣费点。
- 不改变正文 Markdown、语义块结构或复制协议。
- 本轮不增加更多主题，也不引入外部字体或远程样式依赖。
