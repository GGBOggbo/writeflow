# 正文模块化编排管线设计

## 目标

正文创作与公众号排版分成两个明确阶段：

1. 正文模型根据选题、Brief、大纲和素材生成完整纯正文。
2. 独立的“正文模块化编排模块”把纯正文转换为本项目可解析的 GFM Markdown 与 `:::` 高级模块语法。

模块化编排属于正文生成工作流内部步骤，不提供单独按钮，不额外扣积分。只有通过本地语法与内容校验的结果才能进入左侧 Markdown 编辑器和右侧公众号预览。

## 职责边界

### 正文创作模块

- 负责观点、结构、论证、语气、素材槽位和商业落点。
- 输出纯正文 JSON。
- 不负责选择 Markdown 标记或高级排版模块。

### 正文模块化编排模块

- 接收单篇完整纯正文。
- 保留原文事实、观点、案例、数据、人物、结论和行动号召。
- 按手机阅读节奏拆段并添加 GFM 标题、少量强调、引用或列表。
- 根据内容需要选择少量受支持的 `:::` 模块。
- 不发明数字、图片、价格、案例、品牌、人物或身份。
- 输出 Markdown 字符串，不输出 JSON、HTML、解释或代码围栏。

### 本地预览模块

- 只接收通过校验的 Markdown。
- 使用 `renderExtendedMarkdown()` 确定性解析和渲染。
- 不调用 AI，不负责修复模型输出。

## 数据流

```text
正文模型生成纯正文
  -> 保护素材占位符
  -> AI 模块化编排
  -> 恢复素材占位符
  -> 内容完整性校验
  -> Markdown 结构校验
  -> ::: 模块语法校验
  -> 合格：保存为正文版本并进入预览
  -> 不合格：携带具体错误反馈重试一次
  -> 仍不合格：本地转换为基础 Markdown 后进入预览
```

## 模块接口

Provider 接口保持独立语义，命名为 `formatDraftMarkdown`，但扩展为接受可选反馈：

```ts
type DraftModuleLayoutOptions = {
  qualityFeedback?: string;
};

formatDraftMarkdown(
  content: string,
  options?: DraftModuleLayoutOptions
): Promise<string>;
```

Service 层负责重试与校验，Provider 只负责调用模型。

## 校验规则

### 内容完整性

- 输出不能为空。
- 禁止原生 HTML、脚本、样式和 iframe。
- 素材占位符数量、顺序和内容必须保持不变。
- 原文已有高级模块时，模块名、字段键、行宽和围栏必须保持。
- 正文文本覆盖率和长度比例必须达到现有安全阈值。

### Markdown 结构

对于普通长正文，至少满足：

- 存在一个合法 Markdown 标题。
- 存在段落分组。
- 至少出现一种有效节奏结构：粗体、引用、列表、分割线或合法高级模块。

短文可以减少结构要求，但不能把纯文本原样回传并标记成功。

### 高级模块语法

- 模块名必须来自现有 31 个模块目录。
- 每个 `:::` 开始围栏必须有结束围栏。
- 字段型模块必须使用已支持字段结构。
- 行型模块的列数必须稳定。
- 不完整或未知模块视为不合格，不能交给预览。

## 重试策略

第一次输出不合格时，将校验原因转成简洁反馈，例如：

- 缺少 Markdown 小标题。
- 没有任何重点强调、引用、列表或高级模块。
- `:::faq` 缺少结束围栏。
- 使用了不支持的模块名。
- 素材占位符被改写。

第二次请求沿用原始纯正文，并附加“上一次排版不合格，必须修正”的反馈。不能把第一次损坏的 Markdown 当作新的正文继续加工。

## 本地保底

两次 AI 输出均不合格或 Provider 请求失败时，本地确定性生成基础 Markdown：

- 根据原文自然段生成 2-4 个 `##` 小标题。
- 保留全部原文和素材占位符。
- 不生成高级模块，不发明内容。
- 保证 `renderExtendedMarkdown()` 可以解析。

保底结果应记录 `draft.markdown.fallback` 日志和降级进度事件，但用户仍可继续编辑和预览。

## 日志

模块化编排记录：

- 尝试次数。
- 输入、输出字符数。
- 校验失败原因。
- Markdown 标题数、节奏标记数、高级模块数。
- 最终来源：`ai`、`ai_retry` 或 `local_fallback`。

日志不得包含正文全文或提示词全文。

## 测试

- Prompt 测试：初次编排与带质量反馈重试。
- 校验测试：纯文本回显、合法 GFM、合法模块、未知模块、未闭合围栏。
- Service 测试：首次成功、首次失败后二次成功、两次失败后本地保底、占位符保护。
- Provider 测试：反馈被传入第二次 Prompt。
- 集成测试：生成结果必定可由 `renderExtendedMarkdown()` 渲染，且不会出现损坏的 `:::` 原文。

## 非目标

- 不恢复已删除的独立“公众号排版”按钮或 `/api/ai/format` 接口。
- 不新增积分扣费。
- 不让预览器调用 AI。
- 不要求每篇文章强制使用高级模块。
