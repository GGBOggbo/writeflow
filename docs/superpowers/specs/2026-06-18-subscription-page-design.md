# Writeflow 积分与订阅页面设计

## 目标

把已确认的订阅页面原型接入 Writeflow，提供一个真实可访问、使用真实登录用户与积分余额的账户页面。

本阶段只接入前端页面和导航，不接支付、订单数据库、积分充值、会员续费或支付回调。

## 页面入口与路由

- 新增受登录保护的 `/subscription` 页面。
- 工作台右上角的积分卡改为可访问入口，点击进入 `/subscription`。
- 订阅页顶部提供“返回创作工作台”，使用 Next.js `Link` 返回 `/`。
- 继续使用现有 `proxy.ts` 保护该页面；未登录用户访问时跳转 `/login?redirect=/subscription`。

## 数据来源

- 页面使用 Server Component。
- 通过 `safeGetSession()` 获取当前登录用户。
- 通过 `creditStore.getBalance(session.user.id)` 读取真实积分。
- 普通用户展示真实剩余积分。
- 管理员展示 `∞`，并显示“管理员生成不消耗积分”。
- 当前没有会员或订单数据源，因此会员状态固定展示“免费用户”，订单区域展示空状态。

## 页面结构

### 顶部导航

- Writeflow 品牌名。
- 返回创作工作台。
- 当前用户邮箱和简洁头像标识。

### 账户概览

- 当前可用积分。
- 每次生成消耗 1 积分；管理员显示无限积分说明。
- 当前账户状态“免费用户”。
- 不提供“查看积分明细”，因为当前没有独立积分流水产品页面。

### 套餐选择

#### 创作体验包

- `9.9 元`
- `100 创作积分`
- 新用户限购一次。
- 卡片说明：
  - 可用于选题、策略单、大纲、正文和标题摘要生成。
  - 重新生成再次扣除 1 积分。
  - AI 编辑排版与补充素材不消耗项目积分。
  - 生成失败自动退款。
- 本阶段不接支付，因此按钮显示“购买体验包”，但处于不可提交状态，并明确标注“支付功能接入后开放购买”。

#### 月度创作会员

- 价格待定。
- 每月积分和会员权益待定。
- 按钮禁用并显示“即将开放”。
- 不虚构价格、积分或发布日期。

### 订单记录

- 保留订单表格结构，列为购买内容、金额、状态、时间。
- 当前展示“暂无订单记录”。
- 本阶段不创建订单 API 或数据库表。

## 视觉系统

- 使用项目现有蓝灰品牌 token：
  - 主文字 `#233044`
  - 品牌辅助色 `#5f7993`
  - 页面背景 `#f3f5f7`
  - 弱表面 `#eef2f6`
- 页面属于账户工作页，不使用营销落地页式巨型标题。
- 功能容器圆角控制在 `6-8px`。
- 套餐主次通过边框、轻微抬升和按钮层级表达，不使用大面积高饱和底色。
- 数字使用等宽数字特性，确保积分和价格稳定对齐。
- 桌面双列套餐，移动端单列。

## 组件边界

- `app/subscription/page.tsx`
  - 会话读取、积分读取和页面数据装配。
- `components/subscription/subscription-page.tsx`
  - 纯展示组件，接收用户邮箱和积分余额。
- 工作台积分入口
  - 只负责导航，不把订阅页面逻辑塞入 `WorkspaceShell`。

## 状态与错误处理

- `creditStore.getBalance()` 失败时让页面进入现有 Next.js 错误边界，不展示伪造余额。
- 支付和会员按钮不触发请求，不弹出假成功提示。
- 页面不依赖客户端请求完成首屏渲染。

## 测试

- 订阅展示组件：
  - 普通用户展示真实积分。
  - 管理员展示无限积分。
  - 体验包价格和积分正确。
  - 月度会员和购买按钮处于未开放状态。
  - 不出现“积分使用说明”和“查看积分明细”。
- 工作台：
  - 积分卡存在指向 `/subscription` 的入口。
- 页面：
  - 登录会话对应的用户 ID 被用于读取积分。
- 最终运行定向测试、全量测试、lint、build 和桌面/手机浏览器验收。

## 明确不做

- 支付服务商接入。
- 订单、套餐或订阅数据库表。
- 体验包购买资格判断。
- 积分充值写入。
- 支付回调和幂等处理。
- 月度会员价格、积分和权益。
- 积分流水页面。
