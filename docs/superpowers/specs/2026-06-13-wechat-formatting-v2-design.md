# 公众号排版 V2 设计

## 问题证据

2026-06-13 在 Chrome 中检查真实长文排版：100 个内容块中有 98 个 `paragraph`，只有 1 个 `quote` 和 1 个 `cta`。预览高度 4785px，绝大多数段落使用完全相同的字号、颜色和间距。

因此，当前效果差的首要原因不是主题颜色，而是语义识别退化为纯文本。V2 必须先拦截低质量语义结果，再升级渲染视觉。

## 目标

- 长文排版不能以“几乎全是普通段落”的结果成功交付。
- 同一次排版请求允许一次质量重试，不重复扣积分。
- 第二次仍不合格时抛出明确错误，由现有积分链路自动退款。
- 视觉采用“杂志编辑感为骨架，彩色语义块做重点”的方向。
- 继续保证正文逐字不变、只输出公众号兼容的内联样式。

## 语义质量闸门

新增确定性质量分析器，计算：

- 总块数与普通段落比例。
- 非普通语义类型数量。
- quote、pain、transition、cta 等关键角色是否出现。
- 连续普通段落的最大长度。

短文不强求丰富语义。正文达到 20 个块后，如果普通段落比例超过 85%，或特殊语义块少于 3 个，则判定为低质量。正文达到 50 个块后，要求至少 4 个特殊语义块、至少 3 种特殊类型，并限制连续普通段落不能贯穿全文。

质量闸门只判断“结果是否足够表达文章结构”，不修改正文，不自行猜测语义。

## 自动重试

第一次 AI 结果先经过 schema 和正文完整性校验，再经过质量闸门。

若质量不足，服务端记录不含正文的诊断信息，并在同一个请求中再调用一次 provider。第二次 prompt 会携带结构化反馈，例如普通段落占比、缺失类型和最长连续普通段落数，要求模型重新通读全文、识别叙事节点，但仍不得改写文字。

两次调用仍属于同一个 `operationId`，因此只预扣一次积分。第二次仍失败时抛错，流式路由沿用现有退款机制。

## Prompt 策略

- 明确短句不等于普通段落：情绪爆点、结论、反问和停顿要按角色分类。
- 长文必须通读后再标记，不能逐行机械输出 `paragraph`。
- `heading` 只用于原文真实存在的章节标题，不允许凭空添加标题。
- `quote`、`pain`、`transition`、`cta` 可以直接使用原文独立段落。
- 不设置僵硬配额，避免为了过闸门乱标；质量反馈只指出结构缺失。

## 视觉方向

整体采用杂志编辑感：更克制的正文、明确的章节节奏、留白和编辑标签。彩色语义块只用于真正重要的位置。

- 专业蓝：藏青编辑部风格，章节编号、深色金句块、橙色风险语义。
- 温暖橙：暖白纸张、宋体感标题、柔和引用块和暖色提醒。
- 清爽青绿：个人 IP 专栏感，圆角标签、青绿金句块和温暖 CTA。

普通段落减少模板感，不加边框。特殊块不再全部使用同一种浅色方框：金句、痛点、转折、对比和 CTA 各有不同层级。

## 兼容与安全

- 复制内容仍只包含文章 HTML，不含预览外壳。
- 文本继续统一 HTML 转义。
- 禁止 `<style>`、`<script>`、外部资源、复杂定位和 CSS Grid。
- 使用基础块级标签、内联颜色、边框、圆角、间距和安全渐变降级色。

## 验收

- 真实 100 段长文的 98% paragraph 结果会触发重试。
- 第二次合格时整个请求只扣一次积分。
- 第二次仍不合格时返回错误并退款。
- 三套主题渲染测试覆盖新的章节、金句、痛点、转折、对比和 CTA 样式。
- Chrome 中使用同一篇“200 美元续费后受限”的长文重新生成并目视检查。
