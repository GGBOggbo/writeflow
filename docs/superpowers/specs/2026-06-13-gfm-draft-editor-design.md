# GFM 正文阅读与编辑设计

## 目标

正文内容改为受控 GFM Markdown。默认以类似 Typora 的阅读态展示，不显示 `##`、`**` 等源码标记；点击编辑后显示 Markdown 源码，保存后更新当前正文版本。

## 支持范围

- 标题、段落、粗体、斜体、删除线
- 引用、有序列表、无序列表、任务列表
- 链接、图片、分割线
- 表格、行内代码、代码块

不支持原生 HTML、脚本、iframe、样式标签、Mermaid 和数学公式。原生 HTML 作为普通文本处理或从结果中移除。

## 数据与状态

- `DraftVersion.content` 继续使用字符串，但语义调整为 Markdown 源文，不新增数据库字段。
- 新增 `draft_updated` 工作流事件，按 `draftVersionId` 更新正文内容。
- 保存正文后删除该版本的 `draftFormattingByVersion`，防止旧排版继续对应已修改的正文。
- 保存不扣积分；重新执行公众号排版仍按现有规则扣 1 积分。

## 组件

### Markdown 核心

新增统一的 GFM 转换模块：

- 使用 `marked` 解析 GFM。
- 使用 `sanitize-html` 白名单清洗结果。
- 只允许安全链接协议；外部链接补充安全属性。
- 图片只允许安全 URL，保留 `alt` 文本。
- 阅读态和公众号排版共用该模块，避免 Markdown 标记泄漏。

### 阅读态

- 用安全 HTML 渲染正文。
- 采用文章排版样式展示标题、列表、引用、表格和代码。
- 不显示 Markdown 源码标记。

### 编辑态

- 点击“编辑正文”切换为等宽字体 textarea。
- 提供“保存修改”和“取消”按钮。
- 取消恢复进入编辑时的正文。
- 空白正文不能保存。
- 切换正文版本时退出编辑态，避免把内容保存到错误版本。

## AI 提示词

- 正文阶段明确要求在 `content` 字段输出 GFM，不在 JSON 外包裹代码块。
- 使用 Markdown 标题、引用、列表、加粗等表达文章结构；不要滥用格式。
- 去 AI 阶段必须原样保留已有 Markdown 结构和素材占位符，只重写文字。
- 全局规则允许正文阶段按任务要求使用 GFM，其他结构化阶段仍只返回 JSON。

## 公众号排版

- AI 语义分类仍保留原始 Markdown 文本。
- 排版渲染时解析 Markdown，而不是 HTML 转义 Markdown 标记。
- GFM 表格、代码块、列表、链接和图片输出微信兼容的内联样式 HTML。
- 现有语义块（痛点、金句、CTA 等）继续提供外层视觉强调。

## 验证

- GFM 全格式渲染测试与 XSS/危险协议测试。
- 编辑、取消、保存、空内容、切换版本测试。
- 状态机更新正文并清除旧排版测试。
- 提示词要求生成及保留 GFM 的测试。
- 公众号 HTML 不出现未解析的 `##`、`**`，并正确包含表格、代码和任务列表。
