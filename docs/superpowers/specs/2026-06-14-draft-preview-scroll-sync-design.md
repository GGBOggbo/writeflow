# 正文与公众号预览滚动同步设计

## 目标

正文阶段参考 Raphael Publish 的工作台机制，将长正文与公众号预览限制在同一个可视工作区内。两侧各自滚动，并可按阅读进度双向同步，避免两列随着长文一直延伸到页面底部。

## 范围

- 保留现有正文阅读模式和 Markdown 编辑模式。
- 保留现有公众号语义排版、主题切换和复制逻辑。
- 桌面端采用固定高度双栏工作区。
- 左侧正文、右侧预览各自拥有纵向滚动容器。
- 默认开启双向滚动同步，并提供开关。
- 主题切换和复制操作固定在预览面板内，不随文章内容滚动。
- 移动端保持单列自然布局，不强制双栏同步。

不包含：

- 不引入 Raphael Publish 的手机、平板、桌面设备框。
- 不替换现有 Markdown 或公众号 HTML 渲染引擎。
- 不增加点击预览定位 Markdown 源码。
- 不改变正文内容、语义分类或积分逻辑。

## 桌面布局

正文阶段的版本选择区域保持不变。其下方双栏工作区按 iPhone Pro Max
逻辑视口 `430 × 932` 计算可用高度：

- 预览画布保持 `430:932` 的宽高比。
- 根据右栏可用宽度和当前视口可用高度，取较小缩放比例。
- 预览画布最大按 `1:1` 显示，不在宽屏上放大超过真实逻辑尺寸。
- 双栏工作区高度跟随缩放后画布高度，并为顶部工具栏、底部操作区和间距预留空间。
- 左右面板等高。
- 工作区和面板使用 `min-h-0`，保证内部滚动容器可以正确收缩。
- 页面本身不再因完整文章内容无限增高。

左侧面板：

- 顶部固定显示“正文内容”和“编辑正文”。
- 阅读模式中的 Markdown 渲染区独立滚动。
- 编辑模式中的 textarea 占满同一内容区域并独立滚动。

右侧面板：

- 顶部固定显示预览状态、主题切换和滚动同步开关。
- 中间显示等比缩放的 iPhone Pro Max 阅读视窗。
- Pro Max 视窗以 `430 × 932` 为逻辑尺寸，内部文章区域独立滚动。
- 缩放作用于完整阅读视窗，字体、间距、卡片和边框保持同比例。
- 底部固定显示当前版本、复制按钮和复制状态。

## Pro Max 缩放

缩放比例：

```text
scale = min(availableWidth / 430, availableHeight / 932, 1)
```

实现约束：

- 使用 `ResizeObserver` 读取预览容器的实际可用宽高。
- 外层占位尺寸为 `430 × scale` 和 `932 × scale`，参与正常 Flex 布局。
- 内层逻辑画布保持 `430 × 932`，通过 `transform: scale(scale)` 缩放。
- 使用左上角作为变换原点，避免缩放后定位漂移。
- 滚动元素位于逻辑画布内部，因此 `scrollHeight`、`clientHeight` 和滚动比例不受视觉缩放影响。
- 浏览器不支持 `ResizeObserver` 时使用 `scale = 1`，并由外层 `max-width` 防止横向溢出。

## 滚动同步

同步采用 Raphael Publish 已验证的滚动比例策略：

```text
scrollRatio = source.scrollTop / (source.scrollHeight - source.clientHeight)
target.scrollTop = scrollRatio * (target.scrollHeight - target.clientHeight)
```

行为：

- 默认开启。
- 滚动正文时更新预览位置。
- 滚动预览时更新正文位置。
- 使用短暂方向锁避免双方滚动事件互相递归。
- 当一侧没有可滚动距离时，将另一侧同步到顶部。
- 切换正文版本、进入或退出编辑模式时释放方向锁。
- 关闭同步后，两侧完全独立滚动。

同步按阅读进度比例工作，不要求语义块像素级一一对应。两侧排版密度不同，比例同步比直接复制 `scrollTop` 更稳定。

## 组件边界

`DraftStage` 负责：

- 保存同步开关状态。
- 持有正文滚动容器引用。
- 协调正文和预览的双向滚动。
- 切换版本或编辑模式时重置同步锁。

`WechatFormatPanel` 负责：

- 接收预览滚动引用和滚动回调。
- 展示同步开关。
- 将预览内容与底部操作区拆成独立区域。
- 不关心同步比例的具体算法。

同步计算提取为小型纯函数或 Hook，便于独立测试，避免把计时锁和 DOM 计算堆进展示组件。

## 响应式行为

- `xl` 及以上：固定高度双栏和滚动同步。
- 小于 `xl`：恢复单列自然高度；同步开关隐藏或禁用，避免两个纵向滚动容器套在页面滚动中。
- 编辑模式在桌面端沿用固定高度；移动端 textarea 使用合理最小高度。

## 测试

单元测试覆盖：

- Pro Max 缩放比例使用宽高约束中的较小值。
- 缩放比例不超过 `1`，无效尺寸回退为 `1`。
- 缩放后的外层占位尺寸与逻辑画布一致。
- 滚动比例计算。
- 无滚动距离时回到顶部。
- 同步关闭时不更新另一侧。
- 正文滚动驱动预览。
- 预览滚动驱动正文。
- 方向锁阻止递归更新。
- 面板包含固定高度与内部滚动所需的布局类。
- 进入编辑模式后编辑器仍是同步源。

浏览器验证：

- 预览阅读窗口保持 `430:932` 比例。
- 窄屏或矮屏下等比缩小，不裁切、不横向溢出。
- 宽屏下不超过 `430 × 932` 的真实逻辑尺寸。
- 长文不会把双栏撑到页面底部。
- 两侧均能单独滚动。
- 开启同步时双向联动。
- 关闭同步后互不影响。
- 主题和复制按钮始终留在面板操作区。
- 桌面和窄屏布局均无内容截断。
